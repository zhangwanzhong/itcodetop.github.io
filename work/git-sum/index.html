强制覆盖master分支

最近对 Spring Cloud YES 进行了升级，由于改动太多了，导致了大量的冲突，合并比较费劲。于是想用开发分支强制覆盖master分支，以下是步骤。

解决方案

  • 切换到develop分支下，并保证本地已经同步了远端develop的最新代码。

  • 把本地的develop分支强制(-f)推送到远端master。

  • 切换到旧分支master。

  • 下载远程仓库最新内容，不做合并。

  • 把HEAD指向master最新版本。
